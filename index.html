Develop Effective Data Dictionary and API Documentation

Have you ever experienced bad documentation—such as being oversimplified, written for the wrong audience, or lacking clarity?

Whether you're assembling your child's toy or setting up a camera for a once-in-a-lifetime solar eclipse, poorly written and ambiguous documentation can be wasteful and frustrating. Clearly written field definition reference documentation is even more critical, since the target audience is technical and efficiency is key.

Following is a sample JSON and its corresponding field definitions table from an API reference guide. Compare the ambiguous definitions with the useful, descriptive ones written by a technical writer. Which do you prefer?

Sample JSON

  "addressList": [
